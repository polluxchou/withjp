# 内部后台轻量化改版 — 设计文档

> 日期:2026-06-09
> 状态:已与用户确认
> 范围:子项目 1 / 2(子项目 2「主播专区 Creator Portal」另行立项设计)

## 1. 背景与目标

Creator Guild OS 当前是「深蓝侧栏 + 企业表格风」的管理后台,用户(大阪主播公会运营方)反馈**过于严肃**:UI 风格太正式、文案语气太硬。系统未来由**运营团队 + 主播**共同使用,且目前缺少日文支持。

本期目标:

1. 全站换成**风格 B(轻盈现代 SaaS)+ 纹理方案 A(点阵底纹 + 主色光晕 + 磨砂侧栏)**
2. 文案三语(中/英/日)全量重写,语气自然化
3. 接入日文 locale,为下一期主播专区打好设计 token 与日文文案基础

### 项目拆分决策

| 子项目 | 内容 | 状态 |
|--------|------|------|
| 1. 内部后台轻量化改版 | 风格 B 换肤 + 三语文案 + 日文 locale | **本 spec** |
| 2. 主播专区 Creator Portal | 主播角色体系 + 独立 My Page,风格 A(温暖友好) | 下一期单独设计 |

风格 A/B 同属一个 token 家族(暖橙/紫罗兰两端),子项目 2 复用本期 token 架构。

## 2. 已确认的设计决策

- 「严肃」的含义:UI 风格太正式 + 文案语气太硬(功能本身不动)
- 使用者:运营团队 + 主播都会登录
- 语言:中/英/日三语并存,文案三语一起重写
- 风格分流:主播页面用 A(温暖友好),内部管理页用 B(轻盈 SaaS)——本期只做 B
- 实施路线:设计 token 先行,全站一次换肤
- 纹理:方案 A(点阵底纹 + 主色光晕 + 磨砂侧栏)

可视化讨论的 mockup 存档于 `.superpowers/brainstorm/94686-1781065961/content/`(已 gitignore,仅本地参考)。

## 3. 视觉规范(风格 B + 纹理 A)

### 3.1 设计 token

| Token | 值 | 用途 |
|-------|-----|------|
| `bg` | neutral-50 `#fafafa` | 页面底色 |
| `surface` | white + 1px zinc-200 边框 | 卡片/表格容器 |
| `primary` | violet-600 `#7c3aed` | 主色(替换 indigo-600) |
| `primary-soft` | violet-100 `#ede9fe` | 选中态/高亮底 |
| `text` | zinc-900 `#18181b` | 正文 |
| `text-muted` | zinc-400 `#a1a1aa` | 辅助文字 |
| `success-soft` | green-100 底 + green-700 字 | 状态 pill |
| `warning-soft` | yellow-100 底 + yellow-700 字 | 状态 pill |
| `danger-soft` | red-100 底 + red-700 字 | 状态 pill |

圆角:卡片 12px / 按钮 8px / Badge 全圆 pill。
阴影:边框为主,叠极淡阴影 `0 1px 2px rgba(0,0,0,.03)`,拒绝厚重投影。

### 3.2 纹理(方案 A)

- **点阵底纹**:页面底 `radial-gradient(#d4d4d8 0.8px, transparent 0.8px)`,background-size 约 14-16px,极淡
- **主色光晕**:页面顶部一抹紫色径向渐变光晕(`rgba(124,58,237,.14)` 椭圆,向下淡出)
- **磨砂侧栏**:侧栏半透明白 + backdrop-blur,右侧 1px 边框分隔

### 3.3 关键视觉变化

1. 侧栏:深蓝 `#0f172a` → 浅色磨砂,选中项用 primary-soft 圆角块
2. 表格:去全包边框网格 → 行间极淡分隔线,状态列统一 pill 化
3. KPI 卡片:圆角白卡 + 趋势小字(如「↑ 今月 +3人」)
4. 页头问候:按时段显示问候语(おはようございます 👋 等)
5. 空态:emoji + 友好引导文案,替代 "No data"
6. 主色:indigo-600 → violet-600

## 4. 组件改造范围

### 4.1 Token 层(新建,一次性)

- `tailwind.config.ts`:扩展 violet 主色族、zinc 中性族、柔和状态色、圆角、双层柔和阴影
- `globals.css`:CSS 变量 + 三个纹理工具类 `bg-dot-grid` / `bg-glow` / `sidebar-frosted`

### 4.2 共享组件(吃 token,全站自动生效)

- Sidebar:浅色磨砂改造
- Button / Badge / Modal / 输入框 / Select:换 token,Badge pill 化
- 表格样式:行分隔线方案
- KPI 卡片:统一组件化(含趋势小字)
- 新增:**页头问候组件**(按时段)、**友好空态组件**(emoji + 引导文案)

### 4.3 页面清扫

- 全部 24 个模块页面中硬编码的 `slate-*` / `indigo-*` class 替换为 token
- Recharts 图表色板同步换成新色族

### 4.4 非目标(明确不做)

- 不动业务逻辑、API、数据库
- 不做深色模式(本期单亮色)
- 不改布局结构与导航信息架构
- 不做主播专区(子项目 2)

## 5. 文案语气 + 日文化

### 5.1 日文 locale 接入

- 新增 `messages/ja.json`,与 zh/en 同结构
- `next-intl` 路由加 `ja` locale(URL 形如 `/ja/creators`),语言切换器加日文选项
- `check-i18n` / `check-no-bare-han` 校验范围纳入 ja

### 5.2 三语语气规范

| 语言 | 受众 | 语气 | 示例 |
|------|------|------|------|
| 中文 | 团队 | 自然口语,去 ERP 腔 | 「暂无数据」→「还没有记录,添加第一条吧」 |
| 日文 | 团队 + 未来主播 | です/ます体 + 适度亲和 | 「順調です」「あと2回!」 |
| 英文 | 备用 | casual professional,跟随中文语气 | — |

### 5.3 状态枚举口语化

只改显示文案,**不改枚举值**。例:`live` → 配信中/直播中;`onboarded` → デビュー準備/筹备中。

## 6. 实施切分与验证

### 6.1 两个 PR

1. **PR 1:token + 共享组件** — 全站立即换上新皮肤,出问题好回滚
2. **PR 2:页面清扫 + 三语文案 + ja locale** — 机械量大但低风险

### 6.2 验证方式

- `npm run test:copy`(i18n key 同步 + 裸汉字检查)及现有全部单测通过
- 人工核对清单:24 个模块页面浏览器过一遍,三语各抽查重点页(dashboard / creators / expenses / finance-forecast)

## 7. 风险

- 共享组件改造影响面大 → 由 PR 1 单独承载,先回归再继续
- 日文文案质量(です/ます体 + 行业用语 ライバー/配信枠 等)→ 上线前建议由日语母语者抽查
- 24 页清扫易漏 → 以 `grep slate-|indigo-` 清零作为完成标准
